Problem Statement

Modern fintech products need security that people can trust and teams can explain clearly.

Authentication is often presented as a black box. This project turns the security flow into something visible, interactive, and deployable so the product story and the technical story can be demonstrated together.

Idea

Create one platform that combines product UX, backend observability, and security education.

QuantumPay connects a customer-facing fintech website, a realtime operational demo, and an attack lab so one project can showcase the business idea, the implementation logic, and the threat model.

Solution

How QuantumPay solves the problem

  • Registers users with a modern onboarding experience.
  • Generates a per-user secret and derived public identity.
  • Uses fresh challenges for every login attempt.
  • Streams key events to a separate realtime monitoring website.
Outcome

What this platform demonstrates

  • Fintech-oriented user experience.
  • Visible backend security workflow.
  • Attack-defense storytelling for demos and presentations.
  • Deployment-ready separation between backend and static sites.
Tech Stack
Backend Flask, Flask-CORS, Flask-SocketIO, eventlet

Frontend HTML, CSS, JavaScript, CryptoJS, Socket.IO CDN
